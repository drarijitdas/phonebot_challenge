# Phase 5 — UI Design Contract

> Visual and interaction contract for CLI console output. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Note:** This project has no web UI. "UI" refers exclusively to Rich console output rendered in the terminal. The design contract governs Rich markup colors, table structure, typography (terminal font metrics), spacing (blank lines, separators), copywriting (headings, labels, error messages), and interactive prompts. There is no HTML, CSS, or component library. shadcn gate: not applicable (Python CLI project).

| Property | Value |

## Spacing Scale

Terminal spacing is measured in blank lines between output blocks, not pixels. The 8-point grid is mapped to line counts for consistent vertical rhythm.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 0 blank lines | Adjacent labels within a single block |
| sm | 1 blank line | Between pipeline startup lines and first major output |
| md | 1 blank line | Between table and next status line (existing run.py pattern) |
| lg | 2 blank lines | Between the accuracy table block and the comparison table block |
| xl | 2 blank lines | Between major sections in compare.py output (accuracy / per-recording diff / winner) |

Exceptions: none. The existing `console.print()` newline rhythm from run.py is the baseline — Phase 5 additions must match it exactly.

## Typography

Terminal output has no configurable font sizes or weights. Typography contract governs Rich markup styles applied via `[bold]`, `[italic]`, and named color tags.

| Role | Rich Style | Usage |
|------|------------|-------|
| Body | plain (no markup) | Default console.print() lines — status messages, file paths |
| Label | `[cyan]` | Table column headers and field names (established in run.py) |
| Heading | `[bold]` | Section titles, table title strings, winner summary line |
| Accent value | `[green]` | Success states, positive metrics, completion messages (established in run.py) |

Source: run.py lines 62–67, 91, 121–122 establish the project color vocabulary. Phase 5 output must use the same palette.

## Color

Rich named colors map to the terminal's ANSI palette. The project has established a three-color system in run.py.

| Role | Rich Tag | Usage |
|------|----------|-------|
| Dominant (60%) | plain / default terminal | All body text, status lines, file paths |
| Secondary (30%) | `[cyan]` | Table column headers, field name labels |
| Accent (10%) | `[green]` | Success completion lines, positive accuracy values, trace counts, Phoenix URL |
| Destructive | `[red]` | Error states, missing prerequisites (Ollama not running, missing API key, insufficient result files), missing transcript warning |

Accent reserved for: pipeline completion line ("Extraction complete in Xs"), trace count line ("N traces sent to Phoenix"), Phoenix URL line, comparison winner declaration line, per-field accuracy values that exceed baseline.

Destructive reserved for: provider prerequisite errors (D-08 fail-fast messages), compare.py "need at least 2 result files" guard, missing transcript error.

Source: run.py lines 62, 67, 74–75, 91, 117, 121–122 establish this vocabulary. No new colors may be introduced in Phase 5 output.

## Copywriting Contract

All copy is terminal-rendered plain text with Rich markup. No HTML. Sentence case throughout.

| Element | Copy |
|---------|------|
| Pipeline startup heading | `[bold green]Phonebot pipeline starting...[/bold green]` (existing — do not change) |
| Model label | `Model: {model_name}` (existing — do not change) |
| Run accuracy table title | `Per-Field Accuracy` (existing — do not change) |
| Comparison table title | `Model Comparison — Per-Field Accuracy` |
| Latency column header | `Avg Latency (s/rec)` |
| Per-recording diff table title | `Per-Recording Differences` |
| Winner summary line | `[bold green]Winner: {model_name} ({overall_pct}% overall accuracy)[/bold green]` |
| Tie summary line | `[bold]Tie: {model_a} and {model_b} both achieved {pct}% overall accuracy[/bold]` |
| Compare startup line | `[bold]Comparing {N} model result files...[/bold]` |
| Comparison written line | `Comparison written to {path}` |
| Error — too few files | `[red]Need at least 2 result files in outputs/ to compare. Run: uv run python run.py --model <name>[/red]` |
| Error — Ollama not running | `[red]Ollama server not reachable at localhost:11434. Start Ollama and retry.[/red]` |
| Error — missing API key | `[red]ANTHROPIC_API_KEY not set. Add it to your .env file.[/red]` |
| Error — unknown model prefix | `[red]Unknown model '{name}'. Use 'claude-sonnet-4-6' or 'ollama:<model>'.[/red]` |
| Empty state — no recordings | `[red]No transcripts found in data/transcripts/. Run transcription first.[/red]` (existing — do not change) |

Source: CONTEXT.md D-08 (fail-fast with clear error), D-11 (Rich table for compare output), D-15 (bold winner summary). run.py establishes the startup and completion copy patterns.

## Table Contracts

Two new tables are introduced in Phase 5. Both use the existing `rich.table.Table` pattern from run.py.

### Table 1: Model Comparison — Per-Field Accuracy (compare.py)

Printed after loading all result files. One column per model, one row per field plus an Overall row.

| Column | Style | Content |
|--------|-------|---------|
| Field | `[cyan]` | `first_name`, `last_name`, `email`, `phone_number`, `[bold]Overall[/bold]` |
| {model_name_1} | `[green]` for best value, plain for non-best | `{pct}%` — e.g. `87%` |
| {model_name_2} | `[green]` for best value, plain for non-best | `{pct}%` |
| Avg Latency (s/rec) | plain | `{float}s` — e.g. `1.42s` |

Highlight rule: for each row, the highest accuracy value is rendered in `[green bold]`. Tied values are both rendered in `[green bold]`.

### Table 2: Per-Recording Differences (compare.py)

Printed after the accuracy table. Shows only recordings where models disagreed on at least one field.

| Column | Style | Content |
|--------|-------|---------|
| Recording | `[cyan]` | Recording ID, e.g. `call_03` |
| Field | `[cyan]` | Field name |
| {model_name_1} | plain | Extracted value or `[dim]null[/dim]` |
| {model_name_2} | plain | Extracted value or `[dim]null[/dim]` |
| Ground Truth | `[cyan]` | Expected value |

If no differences exist between models, print: `[green]Models agreed on all recordings.[/green]`

Source: CONTEXT.md D-12 (per-field accuracy), D-13 (latency), D-14 (per-recording diff), D-15 (winner summary).

## Interaction Contracts

CLI interactions for Phase 5. All interactions are non-interactive (no prompts to user during execution). Error states print to console and exit with a non-zero code.

| Trigger | System Behavior |
|---------|----------------|
| `uv run python run.py --model claude-sonnet-4-6` | Runs pipeline, writes `outputs/results_claude-sonnet-4-6.json`, prints accuracy table |
| `uv run python run.py --model ollama:llama3.2:3b` | Runs pipeline with Ollama model, writes `outputs/results_ollama_llama3.2_3b.json`, prints accuracy table |
| `uv run python compare.py` | Globs `outputs/results_*.json`, prints comparison tables, prints winner, writes `outputs/comparison.json` |
| `uv run python compare.py` with fewer than 2 result files | Prints `[red]Need at least 2...[/red]` and exits 1 |
| Model registry receives unrecognized prefix | Prints `[red]Unknown model...[/red]` and raises `ValueError` (run.py exits non-zero) |
| Ollama not running when `ollama:*` model requested | Prints `[red]Ollama server not reachable...[/red]` and exits non-zero |
| `ANTHROPIC_API_KEY` missing when `claude-*` model requested | Prints `[red]ANTHROPIC_API_KEY not set...[/red]` and exits non-zero |

Source: CONTEXT.md D-08 (fail fast), D-09 (separate runs + compare), D-10 (model-specific result files), D-11 (dual output).
